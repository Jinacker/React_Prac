---- 기술적 특징을 잘아는게 중요하다
1. 컴포넌트 기반으로 UI를 표현한다.

컴포넌트란 = 구성요소 = 화면을 구성하는 UI 구성 요소를 말함.

리엑트는 
Header
Main
Footer
이렇게 나눠서 구성
이걸 미리 만들어서 모듈화 해둬가지고 그냥 갖다 쓰면됨 => 중복 위험 X

마크업 Mark Up: UI 표현

2. 화면 업데이트 구현이 쉽다

업데이트: 버튼 클릭했을때 바뀌는거 => 리액트는 구현이 쉽다 
=> 리엑트는 선언형 프로그래밍이기 떄문 => 과정 생략 목적만 간결하게 명시 Ex) 토마토 파스타 하나 주세요.

=> 변수 값만 바꿔줘도 화면 업데이트하게 가능. => 막 업데이트되는 원리를 구현할 필요가 없다.


3. 화면 업데이트가 빠르게 처리된다

업데이트 3000번 해야되면 3000번 DOM 바뀜 => 매우 비효율적... => 그래서 아래처럼 구성해야됨.

다양한 업데이트 => 동시에 발생한 업데이트 모아둠 => 모은거 한번만 수정 => DOM

이거를 위한 방식을 직접 짜놔야하는데 => 리엑트는 Virtual Dom을 이용해 이 과정을 자동으로 해준다.

Virtual Dom => 대충 업데이트 다 모아두는 장소 => 모아서 DOM 1회만 수정 => 빠르게 수정 가능.
